## 前情提要

两周前，家里考生愁志愿，市面上的 App 要么收费 99 块，要么广告关不掉。于是一怒之下自己动手，从掌上高考公开 API 抓了 30 省三年录取数据，做了一个微信小程序。

第一版只有三个功能：查院校、查分数线、意见反馈。能用，但粗糙。

## 这几天做了什么

### 1. 专业查询 —— 赛博算命器

填志愿最痛的点：你知道想学「计算机」，但不知道哪些学校有。传统做法是点开几百个学校一个一个翻——翻到第 20 个就疯了。

专业查询的逻辑是反过来的：**你先说专业，系统告诉你哪些学校有**。

但这背后有个技术坑。录取数据是「省份→学校→专业」三层嵌套，18,000 条文档里嵌了 40 万条专业记录。直接扫整张表查一个专业名？云函数跑 6,000 条记录直接 60 秒超时。

最后做了个**倒排索引**——把数据重新组织成「专业名→学校列表」的映射，709 个专业每个一条文档，用 `db.RegExp` 在 `_id` 上一发命中。查询从 30 秒降到 0.3 秒。

输入「计算机」三个字，本地立即联想出「计算机科学与技术」「计算机应用技术」「计算机网络技术」四个备选，点一个，所有开设院校按录取分从高到低排好，点开还能看各省分数。

### 2. 专业目录 —— 这专业到底干嘛的

另一个痛点：专业名字越来越魔幻。「智能感知工程」「区块链工程」「低空技术与工程」——名字听着酷炫，到底学什么？出来能干嘛？

找到了阳光高考网（chsi.com.cn，教育部直属）的公开 API，一个一个爬了 777 个专业的详细信息：
- **专业介绍**：培养目标和课程体系
- **就业方向**：具体到岗位名称（比如计算机出来是「软件开发工程师」「WEB 前端开发」「数据分析师」）
- **性别比例**：男生 72%/女生 28%，直观的进度条
- **学科门类**：工学/理学/医学/农学……十几个大类随便筛

这些数据全是官方来源，不是 AI 编的。比那些收费 App 里的「计算机专业好就业」强一百倍。

### 3. 首页 Dashboard —— 终于不丑了

第一版的首页就是几行文字排排站。改成了 2×2 卡片仪表盘：
- 🏫 院校查询（蓝）
- 📊 分数线（绿）
- 🔍 专业查询（橙）
- 📖 专业目录（紫）

每张卡片带 emoji 图标 + 功能描述 + 彩色顶条，点进去的页面统一有标题栏和返回按钮，不像之前有的页面是 tab 有的不是，体验割裂。

### 4. 性能 —— 从「等半天」到「秒开」

之前的小程序加载院校列表要 30 趟网络请求，在 4G 下转圈转到怀疑人生。

现在：
- **云函数合并请求**：5 路并发批量分页，30 趟合成 1 趟
- **客户端分页渲染**：2988 条数据存在模块变量里，`setData` 只放当前页的 30 条
- **专业索引**：60 秒超时的全表扫描 → 0.3 秒的倒排索引查询
- **一键部署**：`seedAll` 云函数自动链式执行，清空→院校→批次线→录取→专业索引，全自动

## 技术栈

| 层 | 技术 |
|----|------|
| 前端 | 微信小程序原生 |
| 后端 | 云函数（Node.js）+ 云数据库 |
| 数据采集 | Python 多线程 + API 爬取 |
| 数据源 | 掌上高考 / 阳光高考（教育部官方） |

## 数据规模

- **2,988** 所院校
- **18,165** 条投档线文档，含 **400,172** 条专业录取记录
- **709** 个唯一专业名，**777** 条专业详情
- 覆盖 2023/2024/2025 三年，30 个省/直辖市

## 为什么开源

高考志愿填报是信息战——越有钱的家庭越能买到更全的数据，普通家庭拿着省招办发的那本厚册子根本翻不过来。

这些数据本来就是公开的，整理好了免费给大家用。
